🔍 研究发现

1. 核心概念
   -----------
   React Hooks 是 React 16.8 引入的革命性特性,
   允许在函数组件中使用状态和其他 React 特性。

   关键突破:
   - 解决了函数组件无状态的问题
   - 提供了更好的逻辑复用机制
   - 替代了复杂的 class 组件和 HOC

   来源: [2026-01-04-105815] React Hooks 深入理解

2. 常用 Hooks 详解
   ----------------

   a) useState - 状态管理
      用途: 在函数组件中添加状态
      使用场景: 组件内部状态管理

      代码示例:
      ```javascript
      const [count, setCount] = useState(0);
      ```

      最佳实践:
      - 保持状态简单
      - 避免过度拆分状态
      - 合理使用默认值

   b) useEffect - 副作用处理
      用途: 处理副作用操作
      使用场景: 数据获取、订阅、DOM 操作

      代码示例:
      ```javascript
      useEffect(() => {
        document.title = `Count: ${count}`;
        return () => {
          // 清理函数
        };
      }, [count]);
      ```

      最佳实践:
      - 始终指定依赖数组
      - 合理使用清理函数
      - 避免无限循环

   c) useContext - 上下文共享
      用途: 跨组件传递数据
      使用场景: 主题、语言、用户信息等全局状态

      优势: 避免 props 层层传递

   来源: [2026-01-04-105815], [2026-01-04-105900]

3. Hook 规则
   -----------
   必须遵守的规则:
   ✓ 只在函数顶层调用
   ✓ 只在 React 函数中调用
   ✓ 使用自定义 Hook 复用逻辑

   违反规则的后果:
   ✗ 状态不一致
   ✗ 难以调试的错误
   ✗ 性能问题

4. 函数组件最佳实践
   -------------------
   基于 React Hooks 的函数组件开发规范:

   设计原则:
   - 保持组件简洁
   - 单一职责
   - 合理使用解构
   - Props 默认值

   性能优化:
   - 使用 React.memo 避免不必要重渲染
   - 使用 useMemo 缓存计算结果
   - 使用 useCallback 缓存函数

   来源: [2026-01-04-105900]
