# /kb-teach - 费曼技巧学习

## 功能
通过角色扮演实现费曼技巧学习:通过向他人(或 AI)讲解概念来验证和深化理解程度。

## 核心理念

### 费曼技巧
1. 选择一个概念
2. 用简单语言讲解(如同教给初学者)
3. 识别理解盲点
4. 回顾和简化

### 麦肯锡学习法
1. 假设驱动学习
2. 80/20 法则(快速掌握核心)
3. MECE 原则(不重不漏)
4. 验证和迭代

## 使用方式

### 讲师模式(你作为讲师)
```
/kb-teach "React Hooks" --role=teacher

你扮演讲师,向 AI 学生讲解概念
```

### 学生模式(AI 作为讲师)
```
/kb-teach "React Hooks" --role=student

AI 讲师教授你,互动式学习
```

### 基于知识条目教学
```
/kb-teach --id=2026-01-04-105815 --role=teacher

基于特定知识条目进行费曼技巧练习
```

### 麦肯锡学习法
```
/kb-teach "React 性能优化" --mode=mckinsey

使用麦肯锡方法系统化学习
```

### 指定难度级别
```
/kb-teach "JavaScript" --role=student --difficulty=beginner

AI 以初学者视角教学
```

## 参数说明

- `topic|id`: 主题名称或知识条目 ID (必需)
- `--role`: 角色选择
  - `teacher`: 你作为讲师(费曼技巧)
  - `student`: AI 作为讲师(互动学习)
- `--mode`: 学习模式
  - `feynman`: 费曼技巧(默认)
  - `mckinsey`: 麦肯锡学习法
- `--difficulty`: 难度级别
  - `beginner`: 初学者
  - `intermediate`: 中级(默认)
  - `advanced`: 高级
- `--duration`: 交互时长(分钟)
- `--focus`: 重点关注领域
- `--examples`: 是否包含示例(默认:是)

## 学习技巧

### 讲师模式技巧

1. **简化语言**
   - 假设听众是 12 岁的孩子
   - 避免专业术语
   - 使用日常类比

2. **识别盲点**
   - 当无法简单解释时,就是理解不够深
   - AI 学生会追问模糊的地方
   - 记录下来,重点学习

3. **举例说明**
   - 每个概念至少举一个例子
   - 使用生活场景类比
   - 对比不同方案

4. **接受反馈**
   - 不要防御性解释
   - 承认理解不足的地方
   - 根据反馈改进

### 学生模式技巧

1. **主动提问**
   - 不要被动接收
   - 问"为什么""如何""什么情况下"
   - 追问细节

2. **请求类比**
   - "能用生活中的例子解释吗?"
   - "这和 XXX 有什么相似之处?"
   - 帮助建立关联

3. **实践应用**
   - 要求具体场景练习
   - 动手写代码
   - 验证理解

4. **主动回忆**
   - 不看资料,用自己的话说
   - 解释给别人听
   - 发现理解漏洞

### 麦肯锡学习法技巧

1. **假设驱动**
   - 先形成假设
   - 再验证假设
   - 迭代改进

2. **80/20 法则**
   - 识别核心 20%
   - 快速掌握 80%
   - 不要追求完美

3. **MECE 原则**
   - 不重不漏分类
   - 系统化思维
   - 全面覆盖

4. **实践验证**
   - 用实际问题测试
   - 发现偏差
   - 调整理解

## 注意事项

### 1. 诚实评估
- 不要假装理解
- 承认不懂的地方

### 2. 深度思考
- 不要急于回答

### 4. 记录反思

- 改进学习方法

- 验证学习效果
